# DigiWallet Developer Guide: কোড এবং লজিক কানেকশন (Bengali)

এই পেজটি কিভাবে কাজ করছে এবং একটি ফাইলের সাথে অন্যটি কিভাবে লিঙ্ক করা, তা নিচে টেকনিক্যালি ব্যাখ্যা করা হলো।

---

## 🔗 ১. ডেটা ফ্লো (Data Flow) - সবচেয়ে গুরুত্বপূর্ণ অংশ
পুরো প্রোজেক্টটি একটি **"Central Data Pattern"** অনুসরণ করে। 

- **উৎস (Source):** `data.ts` ফাইলে সব কন্টেন্ট রাখা আছে।
- **সংযোগ (Linkup):** `HomePage.tsx` ফাইলটি `data.ts` থেকে সব ডেটা ইমপোর্ট করে। 
- **সরবরাহ (Distribution):** `HomePage.tsx` সেই ডেটাগুলোকে ছোট ছোট কম্পোনেন্টে **Props** এর মাধ্যমে পাঠিয়ে দেয়।

**উদাহরণ:**
১. `data.ts` এ আছে: `export const HERO = { title: "Hello", ... }`
২. `HomePage.tsx` এ সেটা ইমপোর্ট হয় এবং এভাবে পাঠানো হয়: `<Hero data={HERO} />`
৩. `Hero.tsx` এ গিয়ে সেটা রিসিভ হয়: `export default function Hero({ data }) { ... }`

**সুবিধা:** আপনি যদি টেক্সট চেঞ্জ করতে চান, তবে আপনাকে কোড লজিক খুঁজতে হবে না, শুধু `data.ts` এ গিয়ে ভ্যালু চেঞ্জ করলেই হবে।

---

## 🏗 ২. কম্পোনেন্টের স্তরবিন্যাস (Hierarchy)

### ক) মেইন প্যারেন্ট: `HomePage.tsx`
এটি হলো ট্রাফিক পুলিশ। এর কাজ হলো:
- **State ম্যানেজ করা:** থিম (Light/Dark) এবং মেনু ওপেন/ক্লোজ করার স্টেট এখানেই থাকে। 
- **Wiring:** অন্য সব কম্পোনেন্টকে সিরিয়াল অনুযায়ী সাজানো।

### খ) শেয়ারড চাইল্ড: `SectionBlocks.tsx`
এটি একটি ইউটিলিটি ফাইল। যখনই আমাদের কোনো সেকশনে একটি কার্ড বা আইকন বক্স দরকার হয়, আমরা এই ফাইল থেকে `MotionCard` বা `IconBox` ইমপোর্ট করি। প্রায় সব সেকশন (Features, Services, Blog) এগুলোর সাথে লিঙ্কড।

---

## 🎨 ৩. স্টাইলিং এবং থিম কানেকশন (`home.css`)
অন্যান্য প্রোজেক্টের মতো ভেরিয়েবলগুলো আলাদা ফাইলে না রেখে সরাসরি `home.css` এর শুরুতে রাখা হয়েছে।

- **Variable Linkup:** CSS ভেরিয়েবলগুলো (যেমন: `--bg`, `--text`) ব্যবহার করে সব কম্পোনেন্ট ডিজাইন করা।
- **Theme Logic:** যখন `HomePage.tsx` এ থিম চেঞ্জ হয়, তখন `Background.tsx` ফাইলে থাকা একটি `useEffect` বডির ক্লাস (`body.light`) টগল করে। এর ফলে `home.css` এর ভেরিয়েবলগুলো অটোমেটিক চেঞ্জ হয়ে যায়।

---

## 🔄 ৪. ইন্টারঅ্যাকশন কিভাবে কাজ করে?

### মেনু এবং ড্রয়ার (Nav & Drawer)
- `Navbar.tsx` এবং `MobileDrawer.tsx` আলাদা ফাইল হলেও এরা একই স্টেট শেয়ার করে। 
- `HomePage.tsx` থেকে `setDrawerOpen` ফাংশনটি এদের কাছে পাঠানো হয়। যখন আপনি মেইন নেভবারে ড্রয়ার ওপেন করেন, তখন সেটি `HomePage` এর স্টেট আপডেট করে, যা আবার মোবাইলের ড্রয়ারটিকে দৃশ্যমান করে।

### স্লাইডিং এবং লোডিং (Hero & Listing)
- **Hero Carousel:** `Hero.tsx` এর ভেতর একটি `setInterval` চলছে যা ৫.২ সেকেন্ড পর পর `data.ts` থেকে আসা পরের স্লাইডটি দেখায়।
- **Skeleton Loading:** `Listing.tsx` ফাইলে একটি ফেক লোডিং লজিক আছে (`window.setTimeout`)। এটি মূলত দেখায় যে ভবিষ্যতে API কল করলে কার্ডগুলো কিভাবে লোড হবে।

## 🛠 এডিট করবেন কোথায়?

| কি পরিবর্তন করবেন? | কোন ফাইলে যাবেন? |
| :--- | :--- |
| কোনো লেখা, লিঙ্ক বা আইকন | `data.ts` |
| কালার, ফন্ট বা স্পেসিং | `home.css` |
| নতুন কোনো সেকশন যোগ করা | `HomePage.tsx` |
| সেকশনের ভেতরের ডিজাইন (যেমন: কার্ডের সাইজ) | ওই নির্দিষ্ট `.tsx` ফাইল (যেমন: `Features.tsx`) |
| কমন বক্স বা আইকন ডিজাইন | `SectionBlocks.tsx` |

## 💡 মনে রাখার মতো টিপস
১. **Props Wiring:** যদি কোনো নতুন ভেরিয়েবল `data.ts` এ যোগ করেন, তবে সেটি অবশ্যই `HomePage.tsx` এর মাধ্যমে সংশ্লিষ্ট কম্পোনেন্টে পাস করতে হবে।
২. **Framer Motion:** অ্যানিমেশনগুলো সরাসরি ট্যাগের ভেতর দেওয়া আছে (যেমন: `initial`, `animate`, `whileInView`)। এগুলো পরিবর্তন করলে অ্যানিমেশনের ধরন বদলে যাবে।

এখন আপনি যদি কোনো নির্দিষ্ট সেকশন এডিট করতে চান, প্রথমে `data.ts` দেখুন। যদি ডিজাইন পরিবর্তন করতে চান, তবে `home.css` এ গিয়ে ওই সেকশনের ক্লাসটি খুঁজুন।
